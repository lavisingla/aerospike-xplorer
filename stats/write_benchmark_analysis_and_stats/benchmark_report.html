Executive Summary

This report analyzes write performance benchmarks for Aerospike, examining the impact of various configurations on throughput and latency. The analysis covers 36 different configurations, varying record sizes (8192, 16384, 32768, 65536B), thread counts (50, 100, 200), and database fill levels (25, 50, 75%).

Key findings:

  • Maximum throughput: 16763 TPS (achieved with 8192.0B records, 50.0 threads, at 25.0% fill)
  • Best p99 latency: 10.01 ms (achieved with 16384B records, 50 threads, at 25% fill)
  • Average p50/p99/p99.9 latencies across all configurations: 8.34 ms / 139.66 ms / 247.32 ms

Key Insights

Lower thread counts perform better for write operations, with 50 threads achieving the highest average TPS of 8046. Higher thread counts may lead to contention and reduced performance.

Record size significantly impacts write latency. 65536B records show the lowest average p99 latency of 55.13 ms, while 32768B records have the highest at 150.27 ms. This represents a 172.6% increase in latency.

Database fill level has a significant impact on write performance. At 25% fill, p99 latency is lowest, while at 75% fill, it increases by 37.8%. Similarly, TPS is highest at 25% fill and decreases by 16.2% at 75% fill. This demonstrates the performance impact of database utilization on write operations.

Write latency shows significant variability. While average p50 latency is 8.34 ms, p99 latency is 16.8x higher at 139.66 ms, and p99.9 latency is 29.7x higher at 247.32 ms. This indicates occasional high-latency outliers that could impact application performance.

There is a weak correlation (-0.14) between throughput and latency, suggesting that it may be possible to optimize for both high throughput and low latency simultaneously with the right configuration.

Recommendations

For maximum write throughput, use 50.0 threads with 8192.0B records. This configuration achieves 16763 TPS with a p99 latency of 18.51 ms. Note that this configuration works best at 25.0% database fill level.

For lowest write latency, use 50 threads with 16384B records. This configuration achieves a p99 latency of 10.01 ms with 12610 TPS. This configuration works best at 25% database fill level.

For a balanced write performance, use 50 threads with 16384B records at 25% fill level. This configuration achieves 12610 TPS with a p99 latency of 10.01 ms.

Record size significantly impacts write performance. For latency-sensitive operations, use 65536B records. For throughput-oriented workloads, use 8192B records. Consider splitting large records or batching small records to optimize performance based on your application's requirements.

The optimal thread count for both throughput and latency is 50. Configure your client applications to use this level of concurrency for write operations.
